Models

1. Introduction#

Models are the classes used related to the databases, they are in models/ folder

2. MongoDB#

We use MongoDB to store user account information and files

2.1 Mongoose#

We use Mongoose for modeling the data, is an Object modeling tool for MongoDB. The reference is here

2.2 Grid-FS#

It is a file storage tool for MongoDB, the reference is here

3. GraphDB#

GraphDB is a graph database supports RDF and SPARQL, the detail is here The servers connects to GraphDB repository cmmp via localhost:7200

3.1 RDF#

Resource Description Framework. You can read details here

3.2 SPARQL#

SPARQL is a powerful query language. You can read details here

3.3 Ontology#

The description of ontology can refer to here

3.4 GraphDB Model#

We develop a tool by an object oriented way which can create GraphDB model and manipulate more conveniently. It create and execute the SPARQL query internally.

3.4.1 Create a Model#

const PrimaryContactModel = GraphDBModel({
first_name: String,
last_name: String,
position: String,
telephone: String,
telephone_ext: String,
email: String,
}, {
rdfTypes: [Types.NamedIndividual, ':primary_contact_test'],
name: 'primary_contact_test'
});

3.4.2 Add new data in GraphDB#

const newPrimaryContact = await PrimaryContactModel(
first_name: "Test",
last_name: "User",
position: "Software Developer",
telephone: "1234567890",
telephone_ext: "123",
email: "testuser@example.com",
})

3.4.3 Search Data#

const all_primary_contact = PrimaryContactModel.find({})
const primary_contacts_with_first_name = PrimaryContactModel.find({first_name: "Test"})
const primary_contact_with_email = PrimaryContactModel.findOne({email: "testuser@example.com"})
